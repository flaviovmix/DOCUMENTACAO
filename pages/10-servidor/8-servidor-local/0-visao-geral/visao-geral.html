Servidor Local — Praticando o Setup do Nexus

0:00 / 0:00

Réplica local do servidor que está sendo montado pro Nexus — mesmo stack (Caddy + Docker + Postgres + backend Spring + frontend estático), rodando aqui na máquina pra praticar antes de tocar a VM. A ideia é repetir cada fase do setup remoto num ambiente descartável, errar à vontade e só depois aplicar lá.

Por que montar local +

Toda mudança no servidor remoto custa caro: SSH lento, medo de quebrar produção, rollback chato. Local é o oposto — derrubar e subir um container leva segundos, dá pra apagar tudo e recomeçar sem pensar.

O objetivo aqui é treinar o fluxo: subir o Caddy, expor o backend, persistir o Postgres em volume, fazer backup, testar restauração. Quando o gesto virar reflexo, a VM remota deixa de ser assustadora.

Roteiro de prática +

A preencher conforme cada fase for replicada:

  • Subir Caddy local servindo HTML estático
  • Subir Postgres com volume persistente
  • Empacotar o backend do Nexus em imagem Docker
  • Rodar o build estático do frontend e servir via Caddy
  • Backup do volume do Postgres + restauração num container novo
  • Quebrar de propósito e recuperar (deletar volume, derrubar rede, etc.)
