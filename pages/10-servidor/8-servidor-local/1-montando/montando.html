Três caminhos pra ter um Linux rodando aqui na máquina, com perfis diferentes: dual boot Windows + Linux nativo entrega o ambiente mais real (mais bagagem, mais trabalho), VirtualBox entrega sandbox isolado pra testar sem medo, e WSL entrega o Linux integrado ao próprio Windows — leve e direto pra rodar Docker.

Windows + Linux nativos (dual boot) +

O Linux vira um sistema operacional do nível do Windows — escolhe qual bootar no início, e dentro dele tudo é nativo. Sem overhead, sem camada de virtualização, sem "atalho de teclado conflita com o host".

A bagagem vem da própria instalação:

  • Particionar o disco sem destruir o Windows
  • Entender GRUB (o bootloader) vs Windows Boot Manager
  • Lidar com drivers (Wi-Fi, GPU) que às vezes não funcionam de cara
  • Configurar Secure Boot, UEFI, ordem de boot na BIOS
  • Manter o relógio do sistema consistente entre os dois SOs

É o caminho mais demorado e o que tem maior chance de "trava de tarde inteira", mas no fim deixa o Linux como ferramenta de verdade — não como "aquele negócio dentro de uma janela".

VirtualBox +

O Linux roda dentro de uma janela do Windows, como qualquer outro programa. Instala o VirtualBox, baixa a ISO do Ubuntu, cria uma VM nova, e em 15 minutos o sistema está de pé.

A vantagem é a liberdade pra errar:

  • Apaga o sistema inteiro? Deleta a VM e cria outra.
  • Quer testar 3 versões de Ubuntu lado a lado? Cria 3 VMs.
  • Snapshot antes de instalar algo arriscado, rollback se quebrar.
  • Zero risco pro Windows do host.

O custo é performance e proximidade — VM tem overhead, atalhos de teclado podem conflitar, GPU não passa direto. Mas pra treinar o setup do servidor, é mais que suficiente: comandos, containers, redes — tudo igual.

WSL — Windows Subsystem for Linux +

A WSL é o jeito oficial da Microsoft de rodar Linux dentro do Windows. A versão atual (WSL 2) usa uma VM ultra-leve com kernel Linux real, gerenciada pela Microsoft — você nunca vê a VM, não configura RAM, não escolhe disco virtual.

Instala com um comando (wsl --install), abre o terminal do Windows, digita wsl, e cai num shell Linux como se fosse mais uma aba do terminal.

A integração com o Windows é o ponto forte:

  • Arquivos do Linux aparecem no Explorer (\\wsl$\)
  • Arquivos do Windows aparecem dentro do Linux em /mnt/c/
  • Comando do Windows roda de dentro do Linux e vice-versa
  • VS Code abre pasta do WSL com debugger e syntax nativos
  • Docker Desktop usa WSL 2 por baixo — quem já roda Docker no Windows já está usando WSL sem saber

Pra rodar containers e treinar o setup do servidor com Docker, isso significa overhead quase zero e tudo integrado.

A desvantagem é que WSL não é "Linux puro do jeito do servidor". Algumas diferenças:

  • systemd só funciona em versões recentes e com config extra
  • Acesso a hardware físico (USB, GPU) é limitado
  • A separação Windows ↔ Linux fica borrada — não tem a sensação de "estar num sistema totalmente outro"
Qual escolher pra começar +

Os três respondem perguntas diferentes:

  • VirtualBox — pra quem quer aprender Linux como sistema (instalar, particionar, ver o GRUB, lidar com a sessão gráfica) num ambiente isolado e sandbox. É o caminho mais didático.
  • WSL 2 — pra quem quer foco em Docker. Docker Desktop já usa WSL por baixo, então containers rodam integrados e rápidos. Menos didático que VirtualBox, mas o mais direto pra esse exercício.
  • Dual boot — pra quando o objetivo virar "usar Linux como SO de uso diário", não só sandbox.

A escolha recomendada pra esse curso é VirtualBox primeiro — combina aprendizado, sandbox isolado e seguro pra errar. WSL 2 fica como alternativa mais leve se o foco for puramente "rodar Docker rápido".
