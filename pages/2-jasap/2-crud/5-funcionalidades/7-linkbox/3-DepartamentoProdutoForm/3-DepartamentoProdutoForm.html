Conceito — O componente LinkBox

O LinkBox é o componente do Jasap pra representar uma FK 1-N vista do lado do filho. Na tela aparece como um input de texto com o nome do registro vinculado + um pequeno botão verde ao lado. O usuário não digita nele — só clica no botão, escolhe no modal e o texto aparece preenchido.

Três peças formam o circuito:

PeçaPapel
LinkBox no Form do filhoO input visual + botão. Guarda a FK num campo hidden e guarda o texto (nome do pai) visível
Select do paiModal que lista os candidatos e chama o callback ao escolher um
Action de callback no Form do filhoRecebe o ID escolhido, grava no bean e atualiza a tela

O LinkBox sozinho não "conversa" com o Select — quem faz a ponte é a sessão HTTP: o LinkBox grava a URL do callback numa chave que o Select consegue ler. Os dois lados combinam a chave via setCallBackID("_PESSOA").

Carregar o nome antes de mostrar

    getFactory().departamento().pesModel().daoSingle(proBean().getPessoa());
}

Se o produto já tem responsável (estamos em update), vai no banco buscar os dados da pessoa vinculada. O daoSingle recebe o objeto pessoa (que já tem o id_pessoa preenchido, graças à delegação da FK no Bean) e enche os outros campos — inclusive o nome_pessoa, que é o que o LinkBox vai exibir.

Se a FK é null (produto sem responsável), pula — não tem o que carregar.

Detalhe importante do setCallBackID: é um sufixo que o LinkBox anexa à chave LinkBox.CALLBACK_URL ao salvar na sessão. O Select da Pessoa lê essa chave — as duas pontas precisam combinar o mesmo sufixo, senão não se encontram. Convenção: use o nome da entidade em maiúsculas, com underline na frente (_PESSOA).
