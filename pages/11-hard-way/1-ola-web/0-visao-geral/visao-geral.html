Visão Geral — Olá Web na unha

0:00 / 0:00

Esta é a primeira fase da trilha. O objetivo não é fazer um site bonito, é fazer a web na unha: escrever um index.html à mão, abrir no browser, e tentar deixá-lo reativo sem nenhum framework. Você vai criar um contador e uma lista que mudam, mexendo no DOM com JavaScript puro. No caminho, vai bater na dor de manter o estado (uma variável) e a tela (o que aparece) em sincronia na mão. Essa dor é exatamente o que o Svelte resolve, e é por isso que a fase termina apontando pra ele.

O que você vai construir +

Uma página única, sem build e sem dependências, que faz duas coisas:

  • Um contador: um número na tela e um botão "+1" que soma 1 a cada clique.
  • Uma lista de itens, cada um com um botão "remover" que tira aquele item da tela.

Tudo versionado num repositório git próprio, criado do zero na pasta 01-ola-web/ da sandbox. Pequeno de propósito: é simples o bastante pra você fazer inteiro à mão, e já é o suficiente pra dor aparecer.

Git — versionar desde o primeiro arquivo +

Antes de escrever qualquer HTML, a pasta vira um repositório. A ideia é pegar o hábito de salvar o histórico desde o começo, não só no fim.

  • git init — transforma uma pasta comum em repositório (cria a área onde o git guarda o histórico).
  • Working tree vs staged — a diferença entre o que você editou no disco e o que está "marcado" pra entrar no próximo commit.
  • git add — move uma mudança da working tree pra área de stage.
  • git commit — congela o que está em stage num ponto do histórico, com uma mensagem.

No fim da fase você terá pelo menos dois commits: um com a página estática e outro com a parte reativa.

HTML como árvore — o DOM +

O browser não lê seu HTML como texto solto. Ele monta uma árvore de elementos na memória, o DOM (Document Object Model). Cada <h1>, <button>, <li> vira um nó dessa árvore, e é nessa árvore que o JavaScript mexe.

O que você vai entender aqui:

  • O que o browser faz ao abrir um arquivo .html (parse → árvore → desenho na tela).
  • Como a estrutura aninhada do HTML (uma tag dentro da outra) vira a hierarquia de nós.
  • Que mudar a tela é, na prática, mudar essa árvore.
JavaScript tocando o DOM +

Com o DOM na cabeça, o JS entra pra mexer nele. Três funções cobrem tudo nesta fase (são exatamente as que aparecem no código da página):

  • Achar um elemento — document.getElementById("numero") devolve o nó pra você manipular.
  • Trocar o conteúdo — numero.textContent = contador reescreve o texto que aparece; lista.innerHTML = "" esvazia.
  • Escutar um evento — botao.addEventListener("click", ...) roda uma função quando o usuário clica.

Com isso já dá pra fazer o contador e a lista funcionarem. O problema não é a falta de ferramenta, é o que acontece quando a tela cresce.

A dor — estado e tela desencontrados +

Aqui mora o ponto da fase inteira. Você tem o estado de um lado (a variável contador, o array itens) e a tela do outro (o número desenhado, os <li> visíveis). Toda vez que o estado muda, você tem que lembrar de atualizar a tela na mão.

  • Somou no contador? Tem que reescrever o numero.textContent.
  • Mexeu no array com splice? Tem que chamar desenharLista() de novo.
  • Esqueceu de atualizar um lugar? A tela mente: mostra um valor, a variável tem outro. É o bug clássico.

Com uma lista que muda, isso vira um vai-e-vem cansativo: reconstruir a lista na mão a cada mudança, ou caçar exatamente o nó a remover. Essa é a dor que justifica o próximo passo.

Aí o framework entra — Svelte +

Depois de sentir a dor na pele, o Svelte aparece como resposta. Em vez de você sincronizar estado e tela na mão, você declara o estado com a rune $state e escreve onde ele aparece, e o framework mantém os dois juntos:

let contador = $state(0);   // o estado, reativo
<!-- no HTML: --> <p>Contador: {contador}</p>
<button onclick={() => contador++}>+1</button>

Mudou contador, a tela se atualiza sozinha, sem textContent nem desenharLista. Isso se chama reatividade, e é exatamente o que doía na mão. Essa é a peça de frontend da stack do Nexus: o SvelteKit. A fase não te ensina Svelte a fundo ainda, ela te faz querer o Svelte.

O passo a passo da fase +

A ordem do exercício, do mais simples ao ponto da dor:

  1. git init numa pasta nova em 01-ola-web/.
  2. Escrever um index.html com um <h1>, um botão "+1" e um número.
  3. JS na unha: clicar soma 1 e atualiza o texto via DOM.
  4. Primeiro commit.
  5. Adicionar uma lista de itens, cada um com botão "remover", e sentir a dor de re-renderizar a lista na mão.

Quando essa dor ficar óbvia, a Fase 1 cumpriu o papel. O comando /exercícios conduz cada passo e marca o progresso no world map.
