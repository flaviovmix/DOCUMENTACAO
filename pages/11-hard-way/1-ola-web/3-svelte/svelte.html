O alívio: Svelte

0:00 / 0:00

Você sentiu a dor: sem framework, é você quem mantém a tela em sincronia com o estado, na mão, sempre. O Svelte resolve exatamente isso. Vamos montar o projeto e refazer o mesmo contador e a mesma lista — e ver a máquina manual evaporar.

O que o Svelte resolve +

Na versão na unha, cada mudança exigia duas coisas: mexer no estado e mandar pintar na tela (textContent, desenharLista()). A ideia do Svelte é reatividade: você declara o estado, escreve ele no HTML, e o framework mantém a tela em dia sozinho. Você cuida do estado; ele cuida da tela.

É a mesma peça que o Nexus usa no front (SvelteKit) — por isso ela entra aqui.

Montar o projeto — scaffold +

Antes de escrever lógica, precisa existir um projeto. O scaffold monta a estrutura inteira de uma vez (pastas, configs, um exemplo que já roda). Você não cria os arquivos um a um.

npx sv create 01-ola-web-svelte

Ele faz perguntas no terminal. Pras escolhas mais simples pra aprender:

  • Template: minimal (o mais enxuto)
  • TypeScript? Não — JavaScript puro, menos ruído
  • Add-ons? Nenhum por agora

O que ele cria, em dois grupos:

PeçaO que éVocê mexe?

svelte.config.js, vite.config.jsConfigs que ligam as ferramentasQuase nunca

Instalar — npm install +

O scaffold só lista as dependências no package.json; baixá-las é um passo separado. Dentro da pasta do projeto:

cd 01-ola-web-svelte
npm install   # (ou só "npm i")

O npm install lê o package.json e baixa tudo pra dentro de node_modules/. Avisos de "low severity vulnerabilities" são comuns e de baixa gravidade — pode ignorar (não rode audit fix --force, que pode quebrar coisas).

Rodar — npm run dev +

Liga o servidor de desenvolvimento:

npm run dev
  • Ele imprime um endereço, quase sempre http://localhost:5173/ — abra no navegador.
  • O terminal fica ocupado segurando o servidor. Pra parar: Ctrl + C.
  • Hot reload: com ele rodando, salvou um arquivo → a tela atualiza sozinha, sem reiniciar.

É exatamente isto que significa "subir o front" de qualquer projeto SvelteKit — inclusive o Nexus. Lá são dois servidores: o front com npm run dev (:5173) e o back com ./mvnw spring-boot:run (:8080).

O contador reativo — +page.svelte +

No SvelteKit, a pasta src/routes/ vira o roteador, e o arquivo +page.svelte direto nela é a página da raiz (/). O + marca arquivos especiais do framework. É lá que escrevemos. Um arquivo .svelte é um componente: junta a lógica (<script>) e a marcação (HTML) num arquivo só.

<script>
    let contador = $state(0);
    let itens = $state(["Maçã", "Banana", "Laranja"]);
</script>

<h1>Olá Web na unha — agora em Svelte</h1>

<p>Contador: {contador}</p>
<button onclick={() => contador++}>+1</button>

<h2>Lista</h2>
<ul>
    {#each itens as item, i}
        <li>{item} <button onclick={() => itens.splice(i, 1)}>remover</button></li>
    {/each}
</ul>

$state(...) — o estado reativo

$state é uma rune: uma palavrinha do Svelte (começa com $) que marca a variável como reativa. Quando ela muda, tudo que a usa na tela se atualiza sozinho. No JS puro a variável era "burra"; $state deixa ela "esperta".

{contador} — mostrar o valor

As chaves { } colam o valor da variável no meio do HTML, ao vivo: mudou contador, esse ponto se repinta. Substitui o numero.textContent = contador da versão na unha.

onclick={() => contador++} — o clique

Você liga o evento direto no elemento, sem getElementById nem addEventListener. O () => contador++ é uma arrow function (função curta): o mesmo que function () { contador = contador + 1 }. E repare: não tem linha de "pintar na tela".

{#each} — a lista

É um laço dentro do HTML: "para cada item de itens, desenhe este <li>". O Svelte vigia o array e adiciona/remove os <li> sozinho. Esse pedacinho substitui tudo aquilo: o for, o createElement, o appendChild, o innerHTML e a desenharLista inteira.

O que evaporou — na mão vs Svelte +

O mapa do que o framework tirou das suas costas:

Na unha (JavaScript)Em Svelte
getElementById— (não precisa achar nada)
addEventListener("click", ...)onclick={...} no elemento
numero.textContent = ...{contador} (atualiza sozinho)
for + createElement + appendChild + innerHTML + desenharLista()um {#each}

O detalhe que fecha a história: remover um item na unha era itens.splice(i,1) + desenharLista(). Em Svelte ficou só itens.splice(i,1). A linha de redesenhar — a que você era obrigado a lembrar — sumiu. Você cuida do estado, o framework cuida da tela.

$state é uma das runes; tem outras como $derived e $effect, pra mais pra frente.

Código completo — +page.svelte +
