Estilo e Input

0:00 / 0:00

Com o contador e a lista reativos prontos, faltava dar uma cara pra página e um jeito de adicionar frutas novas. As duas coisas entram no mesmo src/routes/+page.svelte: CSS scoped e um input ligado ao estado. E os dois reforçam a mesma lição da fase.

Estilo scoped — o <style> do componente +

No Svelte o CSS mora no próprio .svelte, num bloco <style> no fim do arquivo. O pulo do gato: ele é scoped automático — o que você escreve aqui só vale pra este componente. Por baixo, o Svelte injeta uma classe-hash nos elementos do arquivo e amarra as regras nela, então o seu h1 colorido aqui não vaza pro h1 de outra página.

<style>
    h1 { color: #0f766e; }
    .mais { background: #14b8a6; color: #fff; }
</style>

:global(...) — quando precisa sair do componente

Pra mirar algo de fora (ex: o fundo da página, que mora no <body>), usa :global(...), que desliga o scoping só naquela regra.

<style>
    :global(body) { margin: 0; background: #f0f4ff; }
</style>

Compare com a versão na unha: lá o CSS ia num <style> no <head>, global por natureza, e qualquer seletor podia bater em qualquer página. Aqui é o contrário: nasce isolado, e você abre mão disso de propósito, regra por regra, com :global.

Adicionar à lista — input, estado e bind:value +

Pra criar frutas novas, um <input> ligado a um estado. Declara mais um $state pro texto e uma função que adiciona:

<script>
    let nova = $state("");               // o texto do input

    function adicionar() {
        if (nova.trim() === "") return;  // não adiciona vazio
        itens.push(nova.trim());          // muda o estado...
        nova = "";                        // ...e limpa o input
    }
</script>

<input placeholder="nova fruta..." bind:value={nova}
       onkeydown={(e) => e.key === "Enter" && adicionar()} />
<button onclick={adicionar}>adicionar</button>

bind:value — ligação de mão dupla

O bind:value={nova} liga o campo ao estado nos dois sentidos: você digita e nova muda sozinho; o código muda nova (ex nova = "") e o campo limpa sozinho. Na unha isso era trabalho dobrado: ler input.value na hora de usar e escrever input.value = "" pra limpar, na mão, nos dois lados.

A reatividade de novo

adicionar() só faz itens.push(...) e nova = "". O item novo aparece na lista e o campo esvazia sozinhos, sem nenhuma linha pra redesenhar a lista nem pra limpar o input. É a mesma dor da fase resolvida outra vez: você cuida do estado, o Svelte cuida da tela. O onkeydown com e.key === "Enter" deixa adicionar pelo Enter, além do botão.

Código completo — a lógica do +page.svelte +

O <script> e a marcação depois das duas adições (o <style> scoped fica logo abaixo, no mesmo arquivo):

<script>
    let contador = $state(0);                          // o ESTADO, declarado
    let itens = $state(["Maçã", "Banana", "Laranja"]); // a lista, também estado
    let nova = $state("");                             // o texto do input

    function adicionar() {
        if (nova.trim() === "") return;  // não adiciona vazio
        itens.push(nova.trim());          // muda o estado...
        nova = "";                        // ...e limpa o input
    }
</script>

<h1>Olá Web na unha — agora em Svelte</h1>

<p>Contador: {contador}</p>
<button class="mais" onclick={() => contador++}>+1</button>

<h2>Lista</h2>
<input placeholder="nova fruta..." bind:value={nova}
       onkeydown={(e) => e.key === "Enter" && adicionar()} />
<button onclick={adicionar}>adicionar</button>
<ul>
    {#each itens as item, i}
        <li>{item} <button onclick={() => itens.splice(i, 1)}>remover</button></li>
    {/each}
</ul>
