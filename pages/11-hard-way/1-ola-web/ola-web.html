1 — OLÁ WEB NA UNHA

HTML e JS na mão, mexendo no DOM e segurando o estado manualmente — até a dor pedir o Svelte. Toca git, frontend e protocolos.

O que esta fase ensina: a dor de manter tela e estado em sincronia na mão, os conceitos de git, DOM e eventos, o exercício do contador e da lista, e por que isso prepara o Svelte.

VISAO GERAL

HTML e JavaScript puro: um contador e uma lista que mudam na tela mexendo no DOM na mão. O foco é sentir a dor de manter estado e tela em sincronia.

A PÁGINA NA UNHA

Versionar com git (init, commit) e publicar no GitHub. Inclui a pegadinha do README e as duas formas de conectar: HTTPS e SSH.

MANDAR PRO GITHUB

O scaffold, o npm run dev e o mesmo contador reativo. A reatividade mata a dor: você cuida do estado, o framework cuida da tela.

O ALÍVIO: SVELTE

CSS scoped dentro do componente (e :global pra sair dele) e um input ligado ao estado com bind:value. A mesma lição de novo: você cuida do estado, o framework cuida da tela.

ESTILO E INPUT

O trajeto do comando que sobe o projeto: o npm lê o package.json, chama o Vite, que lê as configs, abre a porta 5173 e fica vivo servindo a página. Parada por parada, do Enter até o app na tela.

NPM RUN DEV

O gatilho do clique, do dedo até a função disparar. O servidor já saiu de cena: o onclick que o Svelte ligou no botão, o bind:value que mantém o texto em dia, e o mesmo destino pelo Enter. Tudo roda no navegador.

O BOTÃO ADICIONAR

O mecanismo: itens.push e itens.splice mexem no estado e o {#each} redesenha a lista sozinho. O contraste com a versão na unha, onde a linha do desenharLista era obrigatória. Você cuida do estado, o framework cuida da tela.

ADICIONAR E REMOVER

O passo zero, do nada até o projeto pronto pra rodar: Node e npm, o scaffold com npx sv create, a estrutura gerada, o .npmrc e o npm install baixando o node_modules. Numerado por último, mas vem antes de todos os outros.
