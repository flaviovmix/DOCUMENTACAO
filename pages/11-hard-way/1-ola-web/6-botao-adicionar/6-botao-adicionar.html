O botão adicionar: o gatilho do clique

0:00 / 0:00

A página já montou (foi o trajeto do episódio anterior). Agora a gente segue um clique no botão adicionar, do dedo até a função disparar. Mudança importante de ator: o servidor (o Vite) já saiu de cena. Daqui pra frente tudo roda no navegador, no runtime do Svelte. O foco deste episódio é o gatilho: como o clique vira uma chamada de função. O que a função faz com a lista é o próximo episódio.

Parada 1 — o botão já nasceu com o handler ligado +

No HTML do componente, o botão é só isto:

<button class="adicionar" onclick={adicionar}>adicionar</button>

Quando o Svelte montou a página, ele viu esse onclick={adicionar} e ligou o evento de clique do botão à função adicionar. Você não escreveu getElementById pra achar o botão, nem addEventListener pra pendurar o ouvinte. O Svelte fez essa ponte sozinho, na compilação. O botão já nasce sabendo quem chamar.

Arquivo: src/routes/+page.svelte (o <button class="adicionar">).

Parada 2 — antes do clique, o input já mantém o texto em dia +

O gatilho precisa de algo pra adicionar: o texto digitado. Quem segura esse texto é a variável nova, ligada ao campo:

let nova = $state("");

<input placeholder="nova fruta..." bind:value={nova} ... />

O bind:value={nova} é de mão dupla: a cada tecla que você digita, ele atualiza nova sozinho. Então, no instante em que o clique vier, nova já tem o texto certo. O gatilho não precisa ir buscar o valor do campo: ele já está na variável.

Na versão na unha isso era passo manual: pra saber o que estava digitado, você lia input.value na hora de usar. Aqui o bind:value mantém a variável em dia o tempo todo.

Parada 3 — você clica: o navegador dispara o evento +

Você clica no botão. O clique é um evento do DOM, gerado pelo navegador, não pelo seu código. Como o Svelte registrou um ouvinte naquele botão lá na montagem, o navegador avisa: "clicaram aqui".

Ponto de fidelidade: nada disso sai do navegador. O servidor não fica sabendo do clique, não tem requisição indo pra lugar nenhum. É só o navegador conversando com o código que o Svelte deixou montado na página.

Parada 4 — o Svelte chama adicionar() +

Avisado do clique, o runtime do Svelte executa a função que estava ligada ali: adicionar. É o destino do gatilho. A função mora no <script> do componente:

function adicionar() {
    if (nova.trim() === "") return;  // o portão de entrada
    itens.push(nova.trim());          // (isto é o próximo episódio)
    nova = "";                        // (isto é o próximo episódio)
}

O trajeto do clique chega aqui: a função roda. O que ela faz com a lista (o push) e com o input (limpar) é exatamente o tema do próximo episódio. Aqui a gente para no gatilho.

Arquivo: src/routes/+page.svelte, função adicionar.

Parada 5 — a primeira coisa que a função faz: o portão +

Antes de adicionar qualquer coisa, a função checa:

if (nova.trim() === "") return;

O nova.trim() tira os espaços das pontas. Se sobrar texto vazio (campo em branco ou só espaços), o return encerra a função ali mesmo e nada acontece. É o portão que evita adicionar fruta em branco. Se passou daqui, é porque tem texto de verdade pra adicionar.

Parada 6 — o mesmo destino por outro gatilho: Enter +

O botão não é a única forma de disparar. O input tem um atalho:

<input bind:value={nova} onkeydown={(e) => e.key === "Enter" && adicionar()} />

O onkeydown escuta as teclas no campo. O e.key === "Enter" && adicionar() quer dizer: se a tecla foi Enter, então chama adicionar(). Apertar Enter no campo dispara a mesma função do botão. Dois gatilhos, um destino.

Quem está no comando: o navegador, não o servidor +

Vale fixar, porque é o contraste com o episódio do npm run dev. Lá o ator era o Vite, o servidor, que serviu o JavaScript e ficou de pé. Aqui o Vite só entregou o código e saiu da história. O clique, o evento, a chamada da função, o portão: tudo roda dentro do navegador, no runtime do Svelte que veio junto com a página. O servidor não espera o seu clique. Quem espera é o navegador.
