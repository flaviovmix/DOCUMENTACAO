Visão Geral — The Hard Way

0:00 / 0:00

Esta trilha reconstrói a stack do Nexus do zero, na unha, atravessando as 7 áreas (git, frontend, backend, banco, segurança, protocolos, servidores). A ideia não é construir o Nexus inteiro à mão. É fazer o começo cru de cada camada pra sentir a dor que o framework resolve. Quando a dor fica óbvia, o Svelte, o Spring, o JPA, o Docker aparecem como resposta, e aí fazem sentido de verdade.

A regra do jogo — dor antes da solução +

Cada área da trilha tem o mesmo vai-e-vem: você faz o mínimo cru, bate na parede, e só então traz o framework como alívio.

  • Frontend — HTML e JS na mão, segurando o estado e mexendo no DOM manualmente. A dor de manter tela e estado em sincronia pede o Svelte.
  • Backend — servidor HTTP cru, roteando e parseando requisição na mão. O boilerplate pede o Spring.
  • Banco — SQL e JDBC crus, abrindo conexão e mapeando linha em objeto na marra. A repetição pede o JPA.

Nunca pular direto pro framework. A dor vem primeiro, senão a peça que resolve não faz sentido.

As 7 fases em ordem +

A jornada é linear: cada fase cruza várias áreas e prepara a próxima. Não são trilhas isoladas, é uma jornada só.

Onde o Nexus aparece +

O Nexus não é o ponto de partida, é o destino. Cada framework que entra na trilha é exatamente uma peça da stack real do Nexus:

  • Svelte na Fase 1 (frontend SvelteKit)
  • JPA na Fase 3 (Spring Data + Postgres)
  • Spring na Fase 4 (backend Spring Boot)
  • Auth na Fase 6 (Spring Security, filtros, rotas públicas)
  • Docker + Caddy na Fase 7 (deploy, 3 containers, VM Ubuntu)

No fim, olhar o código do Nexus deixa de ser misterioso: você já sentiu a dor que cada peça resolve.

O que você entrega no final +

Fechando a Fase 7, você tem nas mãos, concreto:

  • Um app web seu, rodando na web (domínio acessível): uma mini-aplicação de notas (criar, listar, apagar).
  • Front Svelte que fala com back Spring que lê e grava no Postgres, com login protegendo seus dados.
  • Tudo empacotado em Docker + Caddy na VM, o mesmo desenho da stack real do Nexus.
  • Um repositório git com o histórico da jornada, fase por fase.
  • O ganho invisível: capacidade de abrir o código do Nexus e entender cada peça, porque você construiu uma versão mínima de cada uma.

Em uma frase: um "Nexus de bolso" no ar, mais a cabeça pra ler o Nexus de verdade.

Como ler esta trilha +

Aqui a ordem importa, diferente de outras trilhas. Cada fase usa o que a anterior construiu, então o caminho é 1 → 2 → 3 → 4 → 5 → 6 → 7, sem pular.

Cada fase é uma sessão de trabalho na sandbox (PROJETOS/ESTUDO/NEXUS - THE HARD WAY/): você faz o cru, sente a dor, traz o framework, e só então segue pra próxima. O comando /exercícios conduz fase a fase e marca o progresso no world map.
