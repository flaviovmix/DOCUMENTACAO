Olá Mundo no React

0:00 / 0:00

O mesmo primeiro passo, agora no React. A ideia não muda: o agente monta o próprio front e prova que o ambiente liga, com "Olá Mundo" na tela. O que muda são as peças. Aqui o React é só a biblioteca rodando dentro do Vite — sem meta-framework por cima — então a ligação entre as peças fica mais à mostra do que no Svelte.

Por que começar pelo Olá Mundo

Mesma razão de sempre: o Olá Mundo não serve pra ser útil, serve pra provar que a base está de pé. Instalação, versão, projeto montado torto, comando que não roda — tudo isso pode falhar antes de qualquer lógica. Se a frase aparece, o agente sabe que pode construir em cima. É dar a partida no carro antes de viajar.

O scaffold — montar o esqueleto

Igual ao Svelte: o agente roda um comando que monta o projeto inteiro de uma vez — pastas, configs, dependências, um exemplo mínimo que já roda. Recebe a casa com paredes e encanamento prontos e só decora por dentro. Montar é um passo separado de escrever a lógica.

A diferença é o que vem no pacote. No Svelte, o scaffold trouxe o SvelteKit (um meta-framework com roteador embutido). Aqui vem só o React + Vite: mais enxuto, e com a fiação aparente.

A mesma separação do Svelte vale aqui — o agente não trata tudo igual:

package.json — a lista de compras

Mesmo papel: o manifesto do projeto, com nome, scripts e dependências. Mas repare numa diferença em relação ao Svelte: aqui o react e o react-dom ficam em dependencies, não em devDependencies.

{
  "name": "react",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "tsc -b && vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "react": "^19.2.6",
    "react-dom": "^19.2.6"
  }
}

O motivo: o React é uma biblioteca que vai junto pro navegador e roda lá em tempo real. O Svelte, por outro lado, é um compilador — some na hora do build, por isso ficava só como ferramenta de desenvolvimento. Detalhe pequeno, mas mostra a filosofia diferente das duas.

node_modules — a infra que só fica lá

Idêntico ao Svelte: pasta gigante gerada a partir do package.json, com centenas de pacotes. O agente nunca abre nem edita. Não entra no controle de versão — dá pra reconstruir só rodando a instalação de novo.

vite.config.ts — ligando o React no Vite

O Vite é a ferramenta que liga o servidor de desenvolvimento e empacota o projeto. Aqui o config só encaixa o plugin do React dentro dele. É bem menor que o do Svelte porque não tem meta-framework pra configurar.

import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
    plugins: [react()]   // pluga o React dentro do Vite
});

index.html — a casca com a caixa vazia

Aqui aparece a primeira diferença forte de estrutura. No React + Vite, o index.html fica na raiz do projeto e é o ponto de entrada de verdade. Ele tem uma caixa vazia e um script que chama o JavaScript:

<body>
    <div id="root"></div>
    <script type="module" src="/src/main.tsx"></script>
</body>

A <div id="root"> começa vazia. É o buraco onde o React vai desenhar tudo. O script aponta pro arquivo que faz essa ligação.

src/main.tsx — a ligação

Esse é o arquivo que amarra o React na caixa vazia do index.html. Ele pega o elemento root e manda o React desenhar o componente principal lá dentro.

import { createRoot } from 'react-dom/client';
import App from './App.tsx';

createRoot(document.getElementById('root')).render(<App />);

No Svelte/SvelteKit, essa fiação fica escondida pelo framework. No React puro, você vê o passo: "pega a caixa, planta o App dentro". É a ligação entre as peças, à mostra.

src/App.tsx — o componente

Esse é o arquivo que o agente edita pra mostrar o Olá Mundo. É um componente: uma função que devolve o que vai aparecer na tela. Aquele HTML dentro do JavaScript chama-se JSX.

function App() {
    return <h1>Olá Mundo</h1>;
}

export default App;

Igual ao Svelte, o conteúdo útil é uma linha só — um título com a frase. A diferença é a forma: no Svelte você escreve a marcação direto no arquivo .svelte; no React, a marcação volta de dentro de uma função, em JSX.

Rodar e ver na tela

O agente roda o script dev. O servidor de desenvolvimento sobe e dá um endereço local, normalmente na porta 5173 (mesma do Svelte, porque os dois usam o Vite). Ele abre o endereço e vê "Olá Mundo" na página. Mexeu no texto e salvou? A tela atualiza sozinha, na hora — o mesmo feedback rápido.

O que esse passo prova

  • O ambiente está montado. Olá Mundo na tela = base de pé.
  • A fiação ficou visível. No React puro você vê as peças se conectarem: a caixa no index.html, a ligação no main.tsx, o componente no App.tsx. No Svelte isso estava embrulhado pelo SvelteKit.
  • Mesmo conceito, formas diferentes. Os dois mostram "Olá Mundo" com uma linha de conteúdo e muita infra em volta. O que muda é a sintaxe e o quanto o framework esconde de você.
