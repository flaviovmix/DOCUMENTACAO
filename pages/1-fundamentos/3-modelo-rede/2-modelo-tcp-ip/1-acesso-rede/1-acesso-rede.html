Camada 1 — Acesso à Rede

0:00 / 0:00

A camada 1 do TCP/IP é a base — combina o que o OSI separava em duas (Enlace + Física) numa só. Cuida de duas coisas: transformar bits em sinal físico (pulso elétrico no cobre, luz na fibra, onda de rádio no Wi-Fi) e entregar quadros entre vizinhos da mesma rede local. Quando o pacote precisa sair pra outra rede, quem assume é a camada 2 (Internet).

Por que enlace e física viraram uma só +

Na prática, quem mexe com hardware de rede — instala cabo, configura switch, troca placa — não separa "isso é enlace, aquilo é físico". Quando o cabo desconecta, sumiram os dois ao mesmo tempo. Quando a placa para, sumiram os dois.

O TCP/IP simplifica porque a vida real já era assim. Em vez de duas camadas teóricas, uma só — chamada Acesso à Rede — que cobre todo o trecho local entre máquinas que estão no mesmo cabo, no mesmo Wi-Fi, ou no mesmo "bairro de switch".

Os dois trabalhos da camada — bits no fio + entrega entre vizinhos +

Primeiro trabalho — bits viram sinal físico. Cada meio carrega bit de um jeito:

  • Cabo de cobre — pulso elétrico
  • Fibra óptica — pulso de luz
  • Wi-Fi — onda de rádio

Em todos os casos, alguém transforma 0 e 1 em alguma coisa que viaja pelo meio físico — e do outro lado, transforma de volta. Quem faz isso é a camada 1.

Segundo trabalho — entregar quadros entre vizinhos da rede local. "Vizinho" aqui significa: máquina no mesmo cabo, ou no mesmo switch, ou no mesmo Wi-Fi. Pra entregar de vizinho a vizinho, a camada usa um endereço diferente do IP — usa o MAC.

MAC address — endereço físico da placa +

O MAC é o endereço físico da placa de rede. Tem 48 bits, escrito em hexadecimal separado por dois-pontos: a4:c3:f0:1f:b2:de.

  • Vem de fábrica — gravado na placa. Cada placa tem um único no mundo
  • Não muda de rede — leva o notebook pra cafeteria, o MAC continua o mesmo. Só o IP muda
  • Vale só na rede local — quando o pacote sai da rede, o MAC não acompanha

Analogia útil: o MAC é como o número do chassi do carro — vem de fábrica, é único, não muda. O IP é como o número da casa onde o carro está estacionado — muda conforme onde está.

Frame Ethernet — o envelope da camada +

Quando a camada 1 vai entregar dados pra um vizinho, ela monta um frame Ethernet. Frame é o envelope dessa camada.

A camada 1 não olha o conteúdo do pacote IP. Não sabe se lá dentro tem HTTP, banco de dado, e-mail. Só sabe que recebeu um pacote IP pra empacotar e mandar pra um MAC específico.

O switch local usa o MAC de destino pra decidir em qual porta dele o frame vai sair. Switch não sabe nada de IP — só olha MAC e porta física.

ARP — descobrindo o MAC pelo IP +

Surge um problema: a camada 1 trabalha com MAC, mas a camada de cima trabalha com IP. Como a camada 1 sabe qual MAC corresponde a um IP?

A solução é o ARP — Address Resolution Protocol. Quando a camada 1 precisa mandar pra um IP local que ela ainda não conhece, ela grita pra rede inteira:

"Quem aqui tem o IP 192.168.1.5? Manda seu MAC"

A máquina que tem aquele IP responde com o MAC dela. A camada 1 guarda isso numa tabela ARP por alguns minutos pra não precisar perguntar de novo.

ARP só funciona dentro da rede local. Pra IPs fora da rede, quem cuida é o roteador.

Limite da camada — quando o pacote sai do bairro +

Analogia do bairro: o switch local é como um carteiro do bairro. Sabe entregar de uma casa pra outra ali mesmo. Quando a carta tem destino fora do bairro, o carteiro do bairro não sabe o caminho — entrega no posto de saída do bairro, que é o roteador.

Em termos técnicos: quando a camada 1 entrega o frame ao roteador, o roteador "abre" o frame, lê o pacote IP lá dentro, e usa o IP de destino pra decidir pra qual rede mandar em seguida. A partir dali é trabalho da camada 2 (Internet).

Detalhe importante: cada salto entre redes é um frame Ethernet novo, com MACs novos. O IP permanece, mas o MAC muda a cada salto. Por isso o MAC não é endereço de internet — só de bairro.

Sintomas típicos de problema na camada 1:

  • Cabo desconectado — óbvio
  • Sem conexão Wi-Fi — sinal de rádio não chega, ou senha errada
  • Velocidade baixa em Wi-Fi perto de micro-ondas — interferência de rádio
  • Driver da placa de rede com problema — software que conversa com o hardware

Pra dev de aplicação, a camada 1 é quase sempre invisível. Você só percebe ela quando dá problema.
