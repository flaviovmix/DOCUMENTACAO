Lista Ligada

0:00 / 0:00

Uma lista ligada (linked list) é uma coleção de nós conectados por ponteiros. Cada nó guarda um valor + um ponteiro proximo apontando pro próximo nó da cadeia. Diferente do array (que reserva 10 slots contíguos na memória de uma vez), a lista ligada cresce sob demanda: cada nó novo é alocado individualmente e enganchado no fim da cadeia. O exemplo abaixo vem da aula 03 de Estruturas de Dados em Java (Leandro Guarino) e implementa as operações adicionar() e get(i) do zero.

1. Construindo a lista — adicionar()

O primeiro widget foca na escrita: cria a lista vazia e chama adicionar() 4 vezes. A cada chamada o debugger entra no método e mostra o if (primeiro == null && último == null) — só a primeira inserção entra no branch que cria o head; as outras enganchain no fim via ultimo.setProximo(novoElemento).

2. Lendo a lista — getPrimeiro, getUltimo, get(i)

O segundo widget assume a lista já construída (os 4 nós que o widget anterior produziu). Aqui o foco são as leituras: getTamanho e getPrimeiro são instantâneos (O(1), só lêem o ponteiro guardado), mas get(i) precisa caminhar do início — note o atual ↓ avançando posição por posição. get(2) dá 2 hops; get(3) dá 3. Cada println de método encadeado vira 2 passos no compare bar: get() devolve o Elemento, e .getValor() chama nele pra ler a String.

Como funciona — nó com valor + ponteiro +

A lista ligada é construída a partir de uma classe simples — o Elemento — que guarda dois campos:

public class Elemento {
    private String valor;
    private Elemento proximo;  // referência para o próximo nó

    // construtor + getters/setters
}

A classe ListaLigada não guarda os nós num array nem em uma estrutura tipo "lista do Java" — ela só mantém 3 ponteiros:

  • primeiro — referência pro primeiro nó da cadeia
  • ultimo — referência pro último (otimização pra adicionar em O(1))
  • tamanho — contador (otimização pra evitar percorrer a lista pra saber o tamanho)

O resto é uma cadeia de referências: o nó "AC" aponta pra "BA", que aponta pra "CE", que aponta pra "DF", que aponta pra null (fim).

adicionar() — engancha no fim em O(1) +

Adicionar no fim é a operação mais barata da lista ligada — gasta tempo constante (O(1)), porque o ponteiro ultimo já aponta pra onde precisamos enganchar:

public void adicionar(String novoValor) {
    Elemento novoElemento = new Elemento(novoValor);
    if (this.primeiro == null && this.ultimo == null) {
        // lista vazia — primeiro e último apontam pro novo nó
        this.primeiro = novoElemento;
        this.ultimo = novoElemento;
    } else {
        // engancha no fim: último.proximo = novo; depois último = novo
        this.ultimo.setProximo(novoElemento);
        this.ultimo = novoElemento;
    }
    this.tamanho++;
}

Sem o ponteiro ultimo, teríamos que percorrer a lista do primeiro até achar o fim — gastando O(n) cada vez. Mantendo esse ponteiro atualizado, é só "linkar e mover" em 2 operações.

get(i) — caminha do início, gasta O(n) +

Pegar o elemento da posição i é o preço da lista ligada: como os nós não estão em memória contígua (cada um foi alocado separadamente, em endereços quaisquer), não dá pra pular direto pra posição N. Precisa começar do primeiro e seguir os ponteiros proximo até chegar lá.

public Elemento get(int posicao) {
    Elemento atual = this.primeiro;
    for (int i = 0; i < posicao; i++) {
        if (atual.getProximo() != null) {
            atual = atual.getProximo();
        }
    }
    return atual;
}

O atual é um ponteiro "andarilho" que começa no primeiro e dá posicao passos. Pra get(0) não dá passo (retorna o primeiro direto). Pra get(3) dá 3 passos. Em arrays, v[3] é cálculo direto de endereço — em lista ligada, são 3 hops de memória.

Lista ligada vs array — quando usar cada +

Cada estrutura otimiza pra um conjunto diferente de operações:

OperaçãoArrayLista Ligada
Acesso por índice v[i]O(1) — endereço calculado diretoO(n) — precisa caminhar
Adicionar no fimO(1) se há espaço; senão O(n) pra redimensionarO(1) — atualiza ponteiro ultimo
Adicionar no meio (posição i)O(n) — desloca todos os seguintesO(n) pra achar a posição, O(1) pra inserir
Remover do inícioO(n) — desloca todosO(1) — só muda primeiro
Tamanho fixo declaradoSim — new int[10]Não — cresce sob demanda
Uso de memóriaCompacto (1 inteiro = 4 bytes)Cada nó tem overhead do ponteiro (~16 bytes extra por nó em Java)

Regra geral: array é melhor pra leitura aleatória (acesso por índice frequente, tamanho conhecido), lista ligada é melhor pra inserções/remoções nas extremidades. Em Java, ArrayList (array dinâmico) e LinkedList (lista duplamente ligada) são as versões "de fábrica" dessas duas estruturas.
