Remover de Lista Ligada

0:00 / 0:00

Remover um nó de lista ligada é mais delicado que adicionar — não é só "tirar o nó", é religar os ponteiros vizinhos pra cadeia continuar coerente. E dependendo de onde o nó está, a forma de religar muda. São 4 casos: lista com 1 elemento, removendo o primeiro, removendo o último, e removendo do meio. O exemplo abaixo vem da aula 04 de Estruturas de Dados em Java (Leandro Guarino) — implementa o remover() que ficou vazio na aula 03 e exercita os 4 casos num programa só.

📌 A página assume a lista já construída com AC → BA → CE → DF (resultado do widget da página Lista Ligada). Aqui o foco é a segunda metade do Programa.java — a parte que remove e adiciona em sequência, passando por todos os 4 casos.

public void remover(String valorProcurado) {
Elemento anterior = null;
Elemento atual = this.primeiro;
for (int i = 0; i < this.getTamanho(); i++) {
if (atual.getValor().equalsIgnoreCase(valorProcurado)) {
if (this.tamanho == 1) {
this.primeiro = null;
this.ultimo = null;
} else if (atual == primeiro) {
this.primeiro = atual.getProximo();
atual.setProximo(null);
} else if (atual == ultimo) {
this.ultimo = anterior;
anterior.setProximo(null);
} else {
anterior.setProximo(atual.getProximo());
atual = null;
}
this.tamanho--;
break;
}
anterior = atual;
atual = atual.getProximo();
}
}

Os 4 casos do remover() +

Diferente do adicionar() que tem só 2 ramos (lista vazia ou não), o remover() tem 4 ramos, porque a religação dos ponteiros depende de onde o nó está na cadeia:

CasoCondiçãoO que acontece
1. Únicotamanho == 1primeiro = null; ultimo = null; — lista vira vazia
2. Primeiroatual == primeiroprimeiro = atual.proximo; — head avança 1 posição
3. Últimoatual == ultimoultimo = anterior; anterior.proximo = null; — tail recua
4. Meiooutro casoanterior.proximo = atual.proximo; — pula o nó na cadeia

Pra decidir qual caso aplicar, o método primeiro caminha pela lista mantendo 2 ponteiros: anterior e atual. Quando atual.getValor() bate com o procurado, ele testa as condições na ordem da tabela acima e executa o ramo correto.

Por que dois ponteiros — anterior e atual +

Pra remover do meio da lista, você precisa que o nó anterior pule por cima do nó removido — apontando direto pro próximo. Mas o nó anterior não sabe sozinho que ele é o anterior — em lista simplesmente ligada, o ponteiro só vai pra frente.

Solução: o algoritmo mantém dois ponteiros caminhando juntos:

Elemento anterior = null;
Elemento atual = this.primeiro;
for (int i = 0; i < this.getTamanho(); i++) {
    if (atual.getValor().equalsIgnoreCase(valorProcurado)) {
        // achou — usa anterior + atual pra decidir o caso
        break;
    }
    anterior = atual;
    atual = atual.getProximo();
}

O anterior "trail" o atual. Quando o atual bate no procurado, o anterior já está apontando pra exatamente quem precisa religar.

Em lista duplamente ligada (cada nó tem ponteiro pro próximo E pro anterior), esse truque do walking-pair não é preciso — o nó atual já sabe quem é o anterior. É uma das razões pela qual LinkedList do Java é duplamente ligada.

Custo do remover() +

O remover() tem custo O(n) no pior caso — porque precisa achar o nó primeiro (busca linear). Depois de achar, a religação dos ponteiros em si é O(1).

Posição do alvoCusto de achar
Primeiro nóO(1) — testa o primeiro e já achou
MeioO(n/2) na média
ÚltimoO(n) — caminhou até o fim
Não existeO(n) — varreu tudo, saiu do loop sem remover nada

Note um detalhe: o algoritmo do exemplo não trata explicitamente o caso "valor não existe". Ele simplesmente termina o for sem mexer em nada. Em produção, seria bom retornar um boolean indicando se removeu ou não (igual o java.util.LinkedList.remove(Object) faz). Mas a aula 04 mantém simples.

Esvaziar e reconstruir — o ciclo completo +

O Programa.java da aula 04 faz uma sequência interessante: remove e adiciona em padrões diferentes pra atravessar os 4 casos, depois esvazia tudo e recria:

// começa com AC, BA, CE, DF
lista.remover("DF");      // caso 3 — último
lista.adicionar("SP");    // engata no fim
lista.remover("BA");      // caso 4 — meio
lista.remover("CE");      // caso 4 — meio (agora CE é meio)
lista.remover("AC");      // caso 2 — primeiro
lista.remover("SP");      // caso 1 — único
lista.adicionar("RJ");    // primeira inserção (lista estava vazia)

Quando remover("SP") roda, a lista tinha só um elemento (SP). O caso 1 zera primeiro e ultimo — a lista volta ao estado inicial vazio. Depois adicionar("RJ") cai no if (primeiro == null && ultimo == null) de novo — mesma situação do primeiro adicionar("AC") lá no início. A lista se reaproveita sem reset explícito — basta que os ponteiros voltem a null.
