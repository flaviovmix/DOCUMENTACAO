A busca linear (ou busca sequencial) é o algoritmo mais simples de procura: percorre o array do início ao fim comparando cada posição com o valor buscado. Achou? Para. Chegou no fim sem achar? Não existe. O exemplo abaixo, retirado da aula 02 de Estruturas de Dados em Java (Leandro Guarino), procura uma sigla de estado dentro de um array de 10 siglas.

Como funciona — passa pelo array do início ao fim +

A busca linear é o algoritmo mais ingênuo possível: pega o primeiro item, compara com o que você procura, e se não for, vai pro próximo. Repete até achar ou até chegar no fim.

Vantagem: funciona em qualquer array, ordenado ou não. Desvantagem: no pior caso (item no fim ou inexistente) precisa olhar todos os N elementos — diz-se que é O(n). Em arrays grandes e ordenados, existem buscas mais espertas (binária, hash) que veremos depois.

break — quando achou, para de procurar +

O break dentro do for faz o loop encerrar imediatamente — não terminada de checar o resto. Sem ele, o programa continuaria olhando os 10 itens mesmo depois de encontrar o valor.

for (int i = 0; i < estados.length; i++) {
    String elemento = estados[i];
    if (elemento.equalsIgnoreCase(siglaBusca)) {
        encontrou = true;
        break;  // sai do for agora
    }
}

É a otimização mais básica de busca: parar assim que achou. Faz diferença grande quando o array é gigante e o valor está perto do começo.

equalsIgnoreCase — por que não usar == +

Em Java, == entre objetos (e String é objeto) compara se são a mesma referência na memória — não se o conteúdo é igual. Por isso a regra é:

ComparaçãoO que fazQuando usar
a == bCompara referências (endereço)Nunca com Strings (resultado imprevisível)
a.equals(b)Compara conteúdo (case-sensitive)Quando "AC" e "ac" são diferentes pra você
a.equalsIgnoreCase(b)Compara conteúdo, ignora maiúscula/minúsculaQuando "AC", "ac" e "Ac" devem bater

No exemplo, equalsIgnoreCase permite o usuário digitar pb, Pb ou PB — todos batem com "PB" no array.

A flag encontrou — carregando o resultado pra fora do loop +

Note que a variável encontrou é declarada antes do loop, e usada depois dele:

boolean encontrou = false;        // antes
for (...) {
    if (...) {
        encontrou = true;          // dentro
        break;
    }
}
if (encontrou == true) { ... }    // depois

Esse padrão — flag boolean fora do loop — é recorrente em buscas e validações. O loop modifica a flag se algum critério bater, e o código após o loop reage ao resultado. Sem isso, não teria como "saber" o que aconteceu lá dentro.

Detalhe estilístico: em Java, if (encontrou == true) é equivalente a if (encontrou). A versão curta é a preferida — booleanos são autoexplicativos no if.
