Medir Antes de Decidir

0:00 / 0:00

No problema anterior a gente fez o layout preencher o espaço com flex. Mas tem caso em que o CSS sozinho não sabe decidir. A regra era: mostrar uma segunda linha de cards, mas só se ela não atrapalhar o gráfico em cima. Isso depende de quanto espaço sobra de verdade, e o CSS não te conta diretamente. A saída é parar de adivinhar e medir.

O limite do CSS puro — o número mágico falha +

A primeira tentativa foi um limite fixo: se a janela passa de 900 de altura, mostra duas linhas. Deu errado. Em algumas telas a segunda linha aparecia e esmagava o gráfico, até sumir com ele.

// chute: nao sabe quanto os cards e o resto ocupam
const linhas = janelaAltura >= 900 ? 2 : 1;

O motivo é simples: 900 é um chute que não enxerga o layout. Não sabe quanto os cards, mais os outros blocos, realmente ocupam naquela tela. Número mágico não mede nada.

As ferramentas de medir +

A virada é medir o tamanho real renderizado. Duas ferramentas. O getBoundingClientRect dá a posição e o tamanho de um elemento na tela, de verdade, depois que o navegador desenhou. O clientHeight dá a altura interna de um elemento.

const caixa = elemento.getBoundingClientRect(); // {top, left, width, height}
const alturaInterna = container.clientHeight;

Com isso você lê o que de fato aconteceu, não o que imaginou.

Orçar o espaço +

Com a medida na mão, a conta vira um orçamento: você tem um total, desconta o fixo, vê o que sobra. O total é a altura disponível. Desconta o que é fixo (a posição onde o gráfico começa, a base da área de cards). O que resta é o espaço livre, o pool.

const total = dashboard.clientHeight;
const topoDoGrafico = chartBlock.getBoundingClientRect().top - dashboard.getBoundingClientRect().top;
const pool = total - topoDoGrafico - baseDosCards;

// abre a 2a linha so se sobrar gráfico confortavel:
const cabe2 = pool - (2 * alturaDoCard + gap) >= GRAFICO_MIN;

A decisão fica objetiva: abre a segunda linha só se o pool, depois de tirar duas linhas de card, ainda deixa o gráfico com pelo menos um mínimo confortável. Acabou o chute, virou conta.

O perigo do loop e o truque do invariante +

Tem uma armadilha em medir e decidir. Se a sua decisão muda o layout, e você mede justamente aquilo que mudou, dá pra entrar num loop: mede, decide, muda, mede de novo, desfaz, e fica oscilando na tela.

O truque é medir uma grandeza que não muda com a decisão. A altura total da janela não muda por causa dos cards. A posição do topo do gráfico não muda quando você adiciona cards abaixo dele. Medindo coisas estáveis, você decide uma vez e fica firme.

Medir istoMuda com a decisão?
Altura total da janelaNão. Seguro.
Posição do topo do gráfico (cards vêm abaixo)Não. Seguro.
Altura da área de cardsSim. Evitar, gera loop.

A regra geral: baseie a decisão em algo invariante à própria decisão.
