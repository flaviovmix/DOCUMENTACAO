Layout Que Preenche o Espaço

0:00 / 0:00

No problema anterior a gente fez um gráfico parar de cortar a própria linha. Agora o foco é a caixa em volta: como ela decide a altura. A jornada vai de altura fixa em pixels, passa por amarrar na janela com vh e clamp, e chega no preenchimento de verdade com a cadeia flex. No fim, duas decisões de dimensionamento: medir por largura ou por altura, e manter cards numa linha só.

O problema da altura fixa +

Quando você define a altura em pixels, o elemento é sempre daquele tamanho. Numa tela alta sobra espaço embaixo e fica um buraco. Numa tela baixa não cabe, e ou corta ou empurra o resto pra fora. Altura fixa não conversa com o tamanho da tela.

.grafico {
    height: 240px; /* sempre 240, em qualquer tela */
}
Amarrar na janela — vh e clamp +

O primeiro passo pra adaptar é a unidade vh. Um vh é um por cento da altura da janela. Então 20vh é sempre um quinto da altura visível, seja qual for a tela. Já adapta, mas é uma proporção fixa, ainda é meio chute.

Pra não exagerar nos extremos, entra o clamp: você dá um piso, um valor preferido e um teto. Em tela gigante não estica demais, em tela minúscula não some.

.grafico {
    /* no minimo 140px, de preferencia 20vh, no maximo 210px */
    height: clamp(140px, 20vh, 210px);
}

Mesmo assim, continua sendo um palpite proporcional. Não sabe quanto espaço de fato sobrou.

Preencher de verdade — a cadeia flex e o min-height zero +

Pra um elemento ocupar exatamente o que sobra, e não um palpite, usa-se o flexbox. O container vira flex em coluna, e o filho que deve crescer recebe flex: 1. Ele estica pra pegar todo o espaço que os irmãos não usaram.

Tem uma pegadinha que pega quase todo mundo: o min-height: 0. Por padrão, um item flex não encolhe abaixo do tamanho do próprio conteúdo. Isso quebra o "preencher", porque o item se recusa a caber e acaba transbordando. Setar min-height: 0 libera ele pra encolher.

.scroll-area {
    display: flex;
    flex-direction: column;
    min-height: 0;        /* libera o encolher */
}
.chart-block {
    flex: 1;              /* ocupa o que sobra */
    min-height: 0;
}

E precisa ser na cadeia inteira: se são vários níveis aninhados, cada nível que repassa o espaço precisa do flex: 1 e do min-height: 0. Quebrou em um nível, a corrente toda para.

Por largura ou por altura, e a linha de cards +

Duas decisões de dimensionamento. A primeira: medir por largura ou por altura. A propriedade aspect-ratio faz a altura sair da largura (ou o contrário). Um card com proporção fixa: você manda a largura, a altura segue sozinha.

.card {
    aspect-ratio: 3.5 / 4.5;       /* altura derivada da largura */
    max-height: clamp(120px, 24vh, 230px); /* mas responde a altura tambem */
}

Se você quer que o card encolha quando a tela fica baixa, o aspect-ratio sozinho não faz, porque ele olha a largura. Por isso entra um teto de altura em vh junto.

A segunda: cards numa única linha. Existe um jeito de grid (auto-fill com minmax) que cria quantas colunas couberem. Ótimo, mas ele quebra pra próxima linha quando passa do espaço. Se a regra é "sempre uma linha só", isso atrapalha. A alternativa foi calcular quantas colunas cabem e desenhar exatamente esse tanto de card.

/* quebra pra 2a linha quando passa: */
grid-template-columns: repeat(auto-fill, minmax(130px, 1fr));

/* sempre 1 linha, com o numero de colunas calculado: */
grid-template-columns: repeat(var(--cols), 1fr);
