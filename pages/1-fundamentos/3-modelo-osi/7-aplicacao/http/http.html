HTTP

O protocolo da web — como navegador e servidor conversam

Sigla, origem com Tim Berners-Lee, ciclo request/response, métodos (GET, POST, PUT, DELETE), anatomia de uma requisição e diferença pra HTTPS.

Como funciona

Códigos numéricos da resposta do servidor. Categorias 1xx a 5xx, os famosos (200, 301, 404, 500), como ler e o que cada faixa significa em log e DevTools.

Status codes

Metadados da mensagem em formato Nome:valor. Categorias (request, response, entity), formato e regras, headers comuns dos dois lados, negociação de conteúdo, customizados e como inspecionar.

Headers

Anatomia do endereço web (scheme, host, port, path, query, fragment), encoding de caracteres especiais, query string vs path params e como o navegador interpreta.

URL / URI

Como o servidor mantém você logado num protocolo stateless. JSESSIONID, Set-Cookie, fluxo de autenticação, duração e o ciclo cookie-vai-cookie-volta entre browser e Tomcat.

Cookies e Sessão

Headers especiais que controlam comportamento. Cache-Control e ETag pra performance (por que Ctrl+F5 é diferente), Content-Type e MIME pra interpretar JSON, HTML, PDF.

Cache e Content-Type

HTTP sobre criptografia. Handshake TLS, certificado digital, autoridade certificadora e por que o cadeado é o que protege cookie de sessão e senha em trânsito.

HTTPS / TLS

Como o navegador empacota um <form>. GET coloca tudo na query string; POST manda no body em application/x-www-form-urlencoded (texto) ou multipart/form-data (arquivos).
