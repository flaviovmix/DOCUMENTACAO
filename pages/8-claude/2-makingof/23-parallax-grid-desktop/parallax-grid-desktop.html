Parallax e Grid no Desktop

0:00 / 0:00

O site do Kaco estava bonito no celular e quebrado no desktop. Dois consertos nesta sessão: os grids de Personagens e Emoções, que estouravam pra largura total da tela (cards gigantes em monitor grande), e uma faixa parallax nova entre as duas seções, com o personagem de fundo, texto por cima e o fundo seguindo de leve o scroll. A faixa passou por três versões: background-attachment: fixed (100% travada), fixed com texto e overlay, e a final, uma camada de imagem com sangria movida por JavaScript, que funciona inclusive no iOS.

O que a gente queria

A versão mobile estava aprovada, então a regra era mexer só no desktop. Problema 1: num monitor de 1920px, as seções de Personagens e Emoções ocupavam 100% da largura, com a foto num canto do card e o texto lá longe no outro. Problema 2 (que virou pedido em três etapas): uma faixa com a imagem do Kaco entre as duas seções, depois "mais alta com um texto sobre o personagem", e por fim "é possível ele seguir levemente o movimento da página?". Essa última frase matou a primeira implementação e forçou a versão final.

Os conceitos

O truque do full-bleed. Dentro de um container com max-width, dá pra estourar um elemento pra tela inteira com width: 100vw + margin-left: calc(50% - 50vw). 100vw é a largura da janela; o calc puxa o elemento pra esquerda exatamente o que falta pra ele começar na borda. Era proposital pro mobile; no desktop virou o problema.

min-width vs max-width. @media (max-width: 600px) é regra "até 600px" (celular); @media (min-width: 900px) é "de 900px pra cima" (desktop). Corrigir o desktop dentro de um bloco min-width garante, por construção, que o mobile não muda nada.

background-attachment: fixed. Prende o fundo na janela em vez do elemento. Dá profundidade de graça, mas é tudo ou nada: o fundo fica 100% parado. E o iOS ignora a propriedade.

Parallax com sangria. Uma camada de imagem maior que a moldura, com sobra em cima e embaixo (como a sangria de uma gráfica), que o JavaScript empurra com transform: translateY conforme a página rola. O fundo anda mais devagar que a página, como cenário de teatro.

Como montamos

1. O grid contido

O conserto é um bloco novo, sem tocar no CSS original:

@media (min-width: 900px) {
    .char-grid {
        width: 100%;
        max-width: 1100px;
        margin-left: auto;
        margin-right: auto;
        grid-template-columns: repeat(4, 1fr);
    }
}

width: 100% derruba o 100vw; max-width: 1100px é o teto; as margens auto centralizam (e anulam o calc do estouro); repeat(4, 1fr) abre 4 colunas pros personagens. O .emocao-grid ganhou um bloco idêntico com repeat(3, 1fr), porque os cards de emoção têm mais texto.

2. A faixa parallax, versão 1

.parallax-strip {
    height: 380px;
    background-image: url('img/personagens/KACO.jpeg');
    background-size: cover;
    background-position: center 30%;
    background-attachment: fixed;
}

cover estica a imagem até cobrir a área; center 30% escolhe a faixa visível; fixed prende o fundo na janela. Funcionou, com as duas limitações já contadas: travado demais e sem iOS.

3. O texto por cima

A faixa virou um section com um .parallax-content dentro (kicker "O PROTAGONISTA", título "Kaco" e um parágrafo). Três peças: um ::before com inset: 0 e background: rgba(15,39,64,0.45) (overlay azul escuro pro texto branco ter contraste), z-index em camadas (imagem 0, overlay 1, texto 2) e flex pra centralizar. A altura subiu de 380px pra 560px e fechou em 720px, a pedido.

4. O parallax de verdade

O fixed saiu. Entrou um div.parallax-bg vazio com sangria:

.parallax-bg {
    position: absolute;
    left: 0; right: 0;
    top: -220px; bottom: -220px;   /* sangria de 220px */
    z-index: 0;
    background-image: url('img/personagens/KACO.jpeg');
    background-size: cover;
    background-position: center;
    will-change: transform;
}

top e bottom negativos fazem a camada vazar 220px pra cada lado; o overflow: hidden da faixa esconde a sobra. O movimento é do script.js:

const maxShift = 200;   // sempre < sangria (220)
let ticking = false;

function update() {
    ticking = false;
    const rect = strip.getBoundingClientRect();
    if (rect.bottom < 0 || rect.top > window.innerHeight) return;
    const progress = (rect.top + rect.height / 2 - window.innerHeight / 2)
                   / (window.innerHeight / 2 + rect.height / 2);
    const clamped = Math.max(-1, Math.min(1, progress));
    bg.style.transform = `translateY(${(-clamped * maxShift).toFixed(1)}px)`;
}

function onScroll() {
    if (ticking) return;
    ticking = true;
    requestAnimationFrame(update);
}

window.addEventListener('scroll', onScroll, { passive: true });

Peça por peça: getBoundingClientRect() mede a posição da faixa em relação à janela naquele instante; o early-return pula o cálculo quando a faixa está fora da tela; progress é a distância entre o centro da faixa e o centro da janela, normalizada (+1 entrando por baixo, 0 no centro, -1 saindo por cima); Math.max/Math.min é o clamp; o sinal negativo no translateY faz o fundo andar mais devagar que a página. O ticking + requestAnimationFrame é o throttle (um frame agendado por vez, mesmo com o scroll disparando dezenas de eventos por segundo) e passive: true avisa que o listener não bloqueia a rolagem. Antes de tudo, matchMedia('(prefers-reduced-motion: reduce)') desliga o efeito pra quem pediu menos animação no sistema.

Decisões e becos

O fixed morreu por um pedido de uso real. O Flávio rolou a página e estranhou o fundo 100% parado. Com background-attachment: fixed não existe "seguir levemente": é travado ou nada. Foi isso que forçou a camada + JS. De brinde, a versão final funciona no iOS, onde o fixed nem existe.

Sangria e movimento andam juntos. Quando veio o "aumenta o movimento", o maxShift subiu de 110 pra 200 e a sangria teve que acompanhar, de 120 pra 220. Se o deslocamento passa da sobra, aparece a borda da imagem. Regra prática: sangria = maxShift + folga.

O beco do teste. Pra validar sem abrir o navegador na mão, geramos screenshots com o Edge headless. Só que o hero usa altura em vh (janela alta esticava tudo) e o headless não rola até âncora (#personagens devolvia print em branco). A saída foi um HTML temporário só com as seções de baixo, apontando pro style.css real.

Resumo

O desktop quebrado era um truque de full-bleed (100vw + margin-left: calc(50% - 50vw)) ótimo no celular e ruim no monitor; a correção foi um bloco min-width: 900px com max-width: 1100px e mais colunas, sem tocar no original. A faixa parallax nasceu com background-attachment: fixed, ganhou texto com overlay e camadas de z-index, e terminou como uma camada com sangria de 220px empurrada por translateY no scroll: getBoundingClientRect pra medir, clamp pra travar, requestAnimationFrame pra não pesar e prefers-reduced-motion respeitado.
