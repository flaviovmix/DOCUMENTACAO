Componentes do site Kaco

0:00 / 0:00

O site do Kaco é estático: HTML, CSS e JavaScript puro, sem framework e sem build. Mesmo assim deu pra montar três pedaços que valem ser lidos peça por peça: um componente de menu que se injeta sozinho, o texto da seção Sobre contornando o desenho, e um player de áudio com a tag <audio> nativa. O foco aqui é o código de cada um.

O componente de menu (kids-toc.js)

A tentação em site estático é copiar e colar. O menu de cima, por exemplo, está copiado em toda página. Pro menuzinho lateral "Nesta página" a gente fez o contrário: um único arquivo que se injeta sozinho. Cada página só precisa de uma linha, <script src="components/kids-toc.js"></script>, e o CSS mora uma vez no style.css.

O arquivo inteiro é uma IIFE (uma função que se executa na hora) pra não sujar o escopo global. Os itens vivem num array único, a fonte de verdade:

var ITEMS = [
    { id: 'sobre',       label: 'Sobre' },
    { id: 'personagens', label: 'Personagens' },
    { id: 'emocoes',     label: 'Emoções' },
    { id: 'rotina',      label: 'Ser Criança' },
    { id: 'familia',     label: 'Família' }
];

O pulo do gato está em como cada link é montado. A função isHome() pega o fim da URL (location.pathname.split('/').pop()) e diz se a página é a home. Com isso, o link se adapta:

var lis = ITEMS.map(function (it) {
    var href = (home ? '#' : 'index.html#') + it.id;
    return '<li><a href="' + href + '">' + it.label + '</a></li>';
}).join('');

Na home o link é âncora local (#sobre) e o clique rola suave; em qualquer outra página vira index.html#sobre, levando o visitante pra posição certa. O build() começa com um guard (if (document.querySelector('.page-toc')) return null) pra não duplicar, cria o botão com os ícones de tags <i> (hambúrguer e X), e o nav já nasce com a classe collapsed pra não piscar aberto antes do JS rodar. Os dois elementos entram com document.body.appendChild — como são position: fixed, a ordem no DOM não importa.

O comportamento gira em torno de uma função só:

function setOpen(open) {
    nav.classList.toggle('collapsed', !open);
    toggle.classList.toggle('active', open);
    toggle.setAttribute('aria-expanded', open ? 'true' : 'false');
}
setOpen(false); // sempre comeca fechado

O clique no botão faz setOpen(nav.classList.contains('collapsed')) (toggle). No clique de um link, se for home dá preventDefault e scrollIntoView({ behavior: 'smooth' }); fora da home deixa navegar. E em tela ≤1440 o menu fecha após o clique. É componente de verdade: muda o array, muda em todas as páginas.

O texto contornando o desenho (shape-outside)

Na seção Sobre, o desenho do Kaco fica de um lado e o texto do outro. A gente queria o texto encostando no contorno do desenho, não no retângulo da imagem.

Primeira tentativa: shape-outside lendo o canal alpha do próprio PNG.

.sobre-img {
    float: left;
    shape-outside: url('img/sobre.png');
    shape-image-threshold: 0.92;
}

O navegador faz o texto contornar só os pixels não transparentes. Detalhe: shape-outside só funciona com a imagem flutuada (float), por isso o container usa display: flow-root (pra conter o float), sem flexbox. Funcionou em cima, mas embaixo o foguete e o skate se esticam pra direita, e as linhas de baixo ficavam retas e longe. O alpha não "fura" um pedaço opaco que avança.

A virada foi trocar a url por um polígono desenhado à mão, em porcentagem da caixa da imagem. Como o polígono pode ser côncavo, o texto entra nos vãos (sob o capacete, acima do foguete):

.sobre-img {
    float: left;
    shape-outside: polygon(
        20% 12%, 40% 2%, 60% 2%, 67% 12%, 72% 30%,
        60% 42%,   /* entra no pescoço/ombro (côncavo) */
        63% 52%, 90% 60%,   /* braço sai pro foguete */
        85% 70%, 62% 75%,   /* entra antes do skate */
        80% 85%, 74% 94%, 38% 97%, 10% 90%, 3% 62%, 5% 40%, 10% 22%
    ) border-box;
    shape-margin: 40px;
}

Dois detalhes que custaram um beco: o border-box no fim do polygon trava o contorno na caixa da imagem — sem ele, ao subir o Kaco com margin-top negativo o contorno desalinhava (o padrão usa a caixa de margem, que a margem negativa distorce). E o shape-margin é a folga entre o contorno e o texto. Lembrar: só funciona servido por http (live-server); em file:// o navegador bloqueia a leitura dos pixels.

O player de áudio (tag <audio> nativa)

O player veio adaptado de um componente da documentação, pegando só o necessário. A base é a tag nativa <audio preload="metadata">: o metadata baixa só a duração no load, e o áudio em si só baixa ao dar play (importante em página com várias faixas).

O detalhe que faz o botão sempre refletir o estado real: a gente não troca o ícone na mão no clique. O clique só dá play/pause; quem troca o ícone são os eventos do próprio <audio>:

btn.addEventListener('click', () => { audio.paused ? audio.play() : audio.pause(); });
audio.addEventListener('play',  () => { btn.textContent = '⏸'; });
audio.addEventListener('pause', () => { btn.textContent = '▶'; });
audio.addEventListener('ended', () => { btn.textContent = '▶'; fill.style.width = '0'; });

A barra e o tempo vêm do evento timeupdate (dispara enquanto toca): calcula a porcentagem e escreve "atual / total". A barra é clicável pra pular, com uma regra de três em cima do getBoundingClientRect:

audio.addEventListener('timeupdate', () => {
    const p = audio.duration ? (audio.currentTime / audio.duration) * 100 : 0;
    fill.style.width = p + '%';
    time.textContent = fmt(audio.currentTime) + ' / ' + fmt(audio.duration);
});
bar.addEventListener('click', (e) => {
    const r = bar.getBoundingClientRect();
    if (audio.duration) audio.currentTime = ((e.clientX - r.left) / r.width) * audio.duration;
});

O resto da versão completa: os botões de 10 segundos usam Math.max(0, ...) e Math.min(duration, ...) como trava; a velocidade muda audio.playbackRate (com stopPropagation no botão pra o "fecha no clique fora" não fechar o menu na hora de abrir); o volume alterna audio.muted e mexe em audio.volume; e o download é só um <a download href="..."> (botão não baixa, o <a> com download sim).

Nos cards da home entra a versão reduzida: só play/pause e a barrinha clicável. Um detalhe de UX: ao tocar um card, o script pausa os outros áudios pra não tocar dois juntos.

btn.addEventListener('click', () => {
    document.querySelectorAll('.home-audio-item audio').forEach((a) => { if (a !== audio) a.pause(); });
    audio.paused ? audio.play() : audio.pause();
});

Resumo

O kids-toc.js vira um trecho repetido em componente que se injeta sozinho, com fonte única de dados (o array ITEMS) e links que se adaptam à página. O shape-outside com polígono côncavo e border-box faz o texto abraçar o desenho do Kaco, depois que o alpha do PNG não deu conta. E o player usa a tag <audio> nativa, escutando os eventos dela pra refletir o estado, numa versão completa (página) e numa reduzida (cards da home). Três pedaços de um site estático escritos uma vez pra serem reusados.
