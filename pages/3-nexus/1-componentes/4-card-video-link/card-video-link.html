card-video-link — card com vídeo e lista de links

Variante do componente card com vídeo em loop autoplay como mídia de fundo, e painel de info contendo uma lista de sub-links (ul.card-links) que entra em cascata com ícones FontAwesome. Para variante com texto descritivo no painel, veja card-video-texto.

script.js

document.querySelectorAll('.card').forEach(function(card) {
    var btn = card.querySelector('.btnMenuCard');
    var info = card.querySelector('.card-info');

    if (btn && info) {
        btn.addEventListener('click', function(e) {
            e.stopPropagation();
            info.classList.toggle('aberto');
            btn.classList.toggle('aberto');
        });
    }

    var cardHref = card.getAttribute('data-href');
    if (cardHref) {
        card.addEventListener('click', function(e) {
            if (e.target.closest('a[href]') || e.target.closest('.btnMenuCard')) return;
            window.location.href = cardHref;
        });
    }

    var video = card.querySelector('.card-img video');
    if (video) {
        card.addEventListener('mouseenter', function() {
            video.play().catch(function() {});
        }, { once: true });

        var pauseBtn = card.querySelector('.btnPauseCard');
        if (pauseBtn) {
            pauseBtn.addEventListener('click', function(e) {
                e.stopPropagation();
                if (video.paused) video.play().catch(function() {});
                else video.pause();
            });
            video.addEventListener('play',  function() { pauseBtn.classList.add('playing'); });
            video.addEventListener('pause', function() { pauseBtn.classList.remove('playing'); });
        }
    }
});
Como usar — pra quem vai aplicar numa página
0:00 / 0:00

Quando usar

Usar quando:

  • O card representa uma seção/área que tem várias páginas filhas (ex: hub de uma trilha)
  • O painel deve servir como "atalho" pras sub-páginas, não como descrição
  • Você tem um .mp4 mostrando movimento + uma lista de 3–10 sub-links

Não usar quando:

  • O painel deve mostrar texto descritivo → use card-video-texto
  • A mídia é estática → use card-image + o conteúdo de texto

O painel troca o .card-resumo por ul.card-links:

FontAwesome

Os ícones <i class="fa-solid fa-..."> precisam do FontAwesome carregado. No layout principal já vem incluído; em demos isoladas, adicione no <head>:

<link rel="stylesheet"
      href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.1/css/all.min.css">

Limite recomendado de links

O CSS pré-define delays de cascata para até 10 itens. Mais que isso, o último item espera ~0.7s pra aparecer e o ritmo perde graça:

.card-info.aberto .card-links li:nth-child(1) { transition-delay: 0.25s; }
/* ...0.30s, 0.35s, 0.40s... */
.card-info.aberto .card-links li:nth-child(10) { transition-delay: 0.70s; }

Pra mais de 10 links, considera usar card-video-texto com link "Ver todos →" ou criar uma página intermediária.

Como funciona por dentro — pra quem quer entender a mecânica
0:00 / 0:00

Cascata dos links

Cada <li> começa fora-de-cena (opacity: 0; translateX(40px)) e entra com fade + slide quando o painel ganha .aberto:

.card-links li {
    opacity: 0;
    transform: translateX(40px);
    transition: opacity 0.3s ease, transform 0.3s ease;
}

.card-info.aberto .card-links li {
    opacity: 1;
    transform: translateX(0);
}

Os delays escalonados (0.25s → 0.70s) criam efeito cascata — links aparecem da esquerda em sequência, dando ritmo à abertura do painel.

Hover de cada link

.card-links li a:hover {
    background: rgba(255,255,255,0.85);
    transform: translateX(4px);
}

Um leve deslocamento horizontal (4px) reforça que aquele item virou clicável — comum em listas de menu.

Padding extra no painel

O .card-info default tem padding: 0 (porque o .card-resumo já é absoluto com inset: 10px). A variante com links precisa de respiro nas bordas:

.card-info:has(.card-links) {
    padding: 10px 10px;
    overflow: hidden;
}

O seletor :has() aplica o padding só quando há .card-links dentro — mantém o .card-resumo sem mudança.

Decisões de design

Por que cascata em vez de aparecer todos juntos?
Aparecer simultâneo dá sensação de "tela carregando" — bagunça visual. Cascata cria narrativa: "este painel se montou sob demanda", reforçando que cada link foi pensado individualmente.

Por que delays começam em 0.25s?
O .card-info tem transition: transform 0.4s. Se os links começassem em 0s, apareceriam no meio do slide do painel — visualmente confuso. 0.25s dá tempo do painel chegar perto da posição final antes.

Por que slide horizontal e não vertical?
O painel inteiro entra da direita (translateX(100%) → 0). Os links seguindo a mesma direção reforçam o fluxo visual. Slide vertical romperia o vetor de movimento.

Por que ícone separado do texto via justify-content: space-between?
Texto à esquerda, ícone à direita. Padrão de menu reconhecível. O ícone funciona como dica do tipo de destino (seta = página linear, código = ref técnica, engrenagem = setup).

Por que :has() em vez de adicionar uma classe modifier?
A presença do .card-links já é o sinal — não precisa de .card-info--with-links redundante. O :has() deixa o HTML mais limpo (sem classes de estado deriváveis).

Comportamento no mobile

  • Sem hover, o painel só abre via tap no menu de 3 bolinhas (.btnMenuCard)
  • Tap em um link interno do painel navega normalmente — propagação até o card é interrompida pelo e.target.closest('a[href]') guard no handler de data-href
  • Cascata dos links acontece igual — não depende de :hover

Em telas estreitas, considera reduzir o número de links visíveis ou usar fontes menores via media query.
