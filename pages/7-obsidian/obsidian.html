# MOC — Modulo Departamento

> **MOC** = Map of Content. Ponto de entrada do vault sobre o modulo Departamento do XTtreinamento.

## Notas do modulo

- [[Fundacao]] — os 3 arquivos raiz: DepartamentoHome, DepartamentoManager, DepartamentoMdFactory
- [[Pessoa]] — entidade departamento.pes e seus 7 arquivos Java
- [[Produto]] — entidade departamento.pro e seus 8 arquivos Java (tem Select)

## Onde vive

- Codigo: XTtreinamento/src/java/br/xt/app/departamento/
- Banco: schema departamento no PostgreSQL PRD_TREINAMENTO
- Registrado em: [[Fundacao#DepartamentoManager]] — pendura no PnlManager
- E irmao de: LaboratorioManager (modulo de estudo paralelo)

## O que o modulo e

Modulo de estudo de CRUD. Foco: entender **sem a camada de dominio "status derivado"** que o Laboratorio tem (aqui nao ha Perfil — so Produto e Pessoa).

- Entidades: **Produto** e **Pessoa** (sem Perfil)
- Relacao: Pessoa N --- N Produto via tabela de associacao (vista na nota [[Pessoa]])
- Sem FotoUp, sem cascade de status
- Tem Select so em Produto (usado pelo detail da Pessoa)

## Mapa mental rapido

DepartamentoManager        <- registra 31 actions, declara F_ACESSO_MODULO
 |
 +- DepartamentoHome       <- sidebar + breadcrumb (Title/MenuItem/MenuInicial)
 |
 +- DepartamentoMdFactory  <- fabrica de models: proModel() + pesModel()
 |
 +- produto/ (8 arquivos)  -> ver [[Produto]]
 +- pessoa/  (7 arquivos)  -> ver [[Pessoa]]

## Pontos-chave do modulo

- **MID** (__DEPARTAMENTO_MID) e a chave de identificacao do item de menu ativo. 0=home, 1=Produto, 2=Pessoa. Usada por Title, MenuItem e MenuInicial.
- **F_ACESSO_MODULO** = permissao XT.PAINEL_CONTROLE.ACESSO_MODULO.DEPARTAMENTO. Registrada via regFun(...) no config().
- **MdFactory** e instanciada com manager e faz **lazy init** dos models (proModel, pesModel). Cada model so nasce quando alguem pedir.
- Actions internas (Title, MenuItem, MenuInicial, Sort, QuickSearch, tabs, Insert, Update, Delete, etc.) sao **inner static classes** que estendem a action "pai" — padrao Jasap.

## Como navegar o estudo

1. Comecar pela [[Fundacao]] — entender como o modulo e "ligado" no sistema.
2. Ir pra [[Produto]] — entidade mais simples, sem detail.
3. Fechar com [[Pessoa]] — entidade que tem detail N-N com Produto (exercita AddExisting, Desvincular, RefreshDetail).

# Integracoes externas — Todoist e WhatsApp

Conversa com Claude sobre como conectar servicos externos ao Claude Code (alem do [[CLAUDE]] que ja cobre o MCP do Notion).

## Todoist

Tres caminhos possiveis:

### 1. MCP do Todoist (recorrente)
Existe MCP server community (ex: @abhiz123/todoist-mcp-server). Configura em C:\Users\flavi\.claude.json igual ao do Notion, com token da API do Todoist.

- **Onde pegar o token:** Todoist web — Settings — Integrations — Developer
- **Quando usar:** se for consultar/editar tarefas com frequencia

### 2. API REST direta (pontual)
Chamar https://api.todoist.com/rest/v2/tasks via WebFetch com Authorization: Bearer <token>.

- **Quando usar:** um dump unico, sem instalar nada

### 3. Export manual
Todoist web — Settings — Backups — baixa CSV/JSON — salva no vault.

- **Quando usar:** snapshot unico, zero configuracao

## WhatsApp

Mais restrito que Todoist porque o protocolo e fechado.

### WhatsApp pessoal / WhatsApp Business (app)
Mesma situacao pros dois — o app Business verde-escuro e so uma camada visual, nao tem API.

> [!warning]- Atencao: dois produtos com nome parecido
> - **WhatsApp Business (app)** — o app gratuito no celular, sem API
> - **WhatsApp Business Platform / Cloud API** — produto pago da Meta, com API oficial

Opcoes:

1. **Export manual por conversa** — abrir conversa — menu — Mais — Exportar conversa — .txt. Salva no vault, eu leio.
2. **MCP community (lharries/whatsapp-mcp)** — usa a lib whatsmeow (mesma do WhatsApp Web). Login por QR code, guarda em SQLite local. **Nao-oficial** — Meta pode banir a conta se detectar.

### WhatsApp Business Platform / Cloud API (oficial)

Caminho oficial, mas com limitacoes importantes:

> [!danger]- Limitacoes do Cloud API
> - **Nao le historico** — so recebe mensagens novas via webhook, a partir do momento que a integracao subir
> - **So recebe de quem te escrever primeiro** — nao da pra ler grupos, broadcasts
> - **Migrar numero = app some** — numero movido pro Cloud API perde o WhatsApp Business app

**O que precisa pra montar:**

1. Conta Meta Business verificada (CNPJ, documentos)
2. Numero migrado pro Cloud API (ou numero novo)
3. **Webhook endpoint** — servidor HTTPS (VPS, Cloudflare Worker, ngrok em dev)
4. Servidor grava mensagens em algum lugar — Claude le de la
5. Opcional: BSP (Twilio, 360dialog, Zenvia) pra simplificar

### Precos (Cloud API, Brasil, ordem de grandeza)

Meta cobra por **conversa** (nao por mensagem):

| Categoria | Valor aprox. |
|-----------|--------------|
| Marketing | ~R$ 0,25 |
| Utility | ~R$ 0,08 |
| Authentication | ~R$ 0,07 |
| Service (cliente inicia, janela 24h) | geralmente gratis |

- **1000 conversas service gratis/mes**
- **BSP** (Twilio etc) cobra markup em cima
- **Setup Meta e gratis**, mas exige Business verificado
