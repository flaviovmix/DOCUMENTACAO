Um Monitor Flutuante

0:00 / 0:00

Making-of de uma extensão do Chrome pro YouTube Studio: durante uma transmissão ao vivo, deixar só dois números na tela (Espectadores e Gostei) pra um narrador monitorar. O valor aqui não é o código final, é o caminho: a gente errou de propósito e corrigiu a estratégia duas vezes. Arquivos reais: manifest.json, content.js e content.css.

A dor

O control room do Studio é cheio: preview, chave de transmissão, chat, barra de navegação na esquerda, cabeçalho em cima. Toda transmissão, a saída era limpar tudo isso na mão pra sobrar só o quadrinho com os números que o narrador acompanha. A ideia foi automatizar num botão. O Studio usa web components, mas em "shady DOM" (DOM normal), então o CSS de fora alcança os elementos.

Primeira abordagem: limpar ao redor

A ideia natural foi esconder peça por peça. Inspeciona o chat e esconde; depois a navegação; depois o cabeçalho. Cada um virava uma regra de CSS ligada por uma classe no <html>:

html.yt-chat-hidden #chat,
html.yt-chat-hidden ytls-navigation,
html.yt-chat-hidden #header-container {
    display: none !important;
}

Funcionava, mas a lista não parava de crescer. O control room tem muito painel, e em estados diferentes da live (offline, ao vivo, iniciando) aparecem coisas novas. Ia ser caça a elemento pra sempre.

Primeira virada: focar no que fica

A correção de rota: em vez de listar tudo que sai, inverter e dizer qual é o único elemento que fica, escondendo o resto. O alvo é o #viewers-row. A função isolate começa nele e sobe a árvore até o body; em cada nível, esconde os irmãos do nó atual. Sobra só o caminho do body até o quadro. O botão é poupado pra não sumir junto:

function isolate() {
    const target = findTarget();
    if (!target) return;
    target.dataset.ytFocusTarget = "1";
    let el = target;
    while (el && el.parentElement && el !== document.body) {
        const parent = el.parentElement;
        Array.from(parent.children).forEach(function (sib) {
            if (sib !== el && sib.id !== BTN_ID) hideEl(sib);
        });
        el = parent;
        if (el !== document.body) el.dataset.ytFocusKeep = "1";
    }
}

O que o YouTube criar depois já nasce escondido, porque o isolate re-roda quando a página re-renderiza (via MutationObserver). Pra grudar no canto e tirar o espaço vazio, o quadro vira position: fixed no canto e os containers do caminho perdem min-width e fundo.

Segunda virada: o muro do tamanho

A meta era encolher a janela até a largura dos dois números. Não dava: parava num ponto. O motivo não era mais a página, era o Chrome. Uma janela normal do navegador tem largura mínima e você não arrasta pra menos. Nenhum CSS resolve, porque é limite da janela, não do site. Reconhecer isso foi o que destravou: não adiantava insistir no CSS, a saída era trocar de ferramenta.

A solução: Document Picture-in-Picture

O Document Picture-in-Picture é uma API do Chrome que abre uma janelinha separada, sempre por cima, redimensionável à vontade. A openMonitor chama requestWindow (só funciona disparada por um clique, que exige gesto do usuário) e monta um HTML próprio dentro dela:

async function openMonitor() {
    if (!window.documentPictureInPicture) throw new Error("Document PiP indisponível");
    pipWindow = await window.documentPictureInPicture.requestWindow({ width: 320, height: 130 });
    // ... monta o style e os dois blocos (label + número) ...
    sync();
    syncTimer = setInterval(sync, 1000);
    pipWindow.addEventListener("pagehide", closeMonitor);
}

O espelho ao vivo

A janelinha não calcula nada. A readStats lê o #viewers-row na aba do Studio, pega os valores e labels, e a sync copia pra janelinha a cada 1 segundo (setInterval). Quem mantém o número fresco é o próprio YouTube. O detalhe que confunde: a gente esconde com display: none, mas isso não pausa o JavaScript nem o data-binding. O número segue sendo atualizado no DOM mesmo escondido, por isso o espelho funciona.

function readStats() {
    const row = document.querySelector(TARGET);
    if (!row) return null;
    return Array.from(row.querySelectorAll(".metadata-item"))
        .filter(function (it) { return !it.hasAttribute("hidden"); })
        .map(function (it) {
            const label = it.querySelector(".label");
            const value = it.querySelector(".value");
            return {
                label: label ? label.textContent.trim() : "",
                value: value ? value.textContent.trim() : "–",
            };
        });
}

O fallback

Se o navegador não suportar o Picture-in-Picture, o clique cai sozinho no modo foco da aba (o isolate de antes). É um try-catch: tenta abrir a janelinha; se der erro, ativa o foco. Nada de tela quebrada. O texto do botão conta o estado:

async function onClick() {
    if (pipWindow) { closeMonitor(); return; }
    if (focusActive) { focusActive = false; applyFocus(); updateLabel(); return; }
    try {
        await openMonitor();        // tenta a janelinha flutuante
    } catch (e) {
        focusActive = true;         // não rolou: cai no modo foco
        applyFocus();
        updateLabel();
    }
}

Resumo

O caminho: começamos escondendo peça por peça (lista que não acabava); invertemos pro modo foco (isolate sobe a árvore e deixa só o quadro, grudado no canto); e batemos no limite de largura da janela do Chrome, o que forçou a virada final, uma janelinha Picture-in-Picture sempre por cima espelhando os contadores ao vivo, com o foco de reserva. As duas coisas pra levar: quando a lista de "esconde isso também" não acaba, inverta e diga o que fica; e quando o limite é da ferramenta (a janela do navegador), troque de ferramenta em vez de insistir no código.
