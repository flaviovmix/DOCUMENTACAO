Escondendo a Lista do WhatsApp

0:00 / 0:00

Este é o making-of de uma extensão do Chrome feita do zero, com um objetivo pequeno e concreto: um botão que esconde a lista de conversas do WhatsApp Web e a traz de volta com outro clique. O foco é o código, peça por peça. Arquivos reais: manifest.json, content.js e content.css.

A raiva de todo dia

Quando você deixa a janela do WhatsApp Web menor, pra caber do lado de outra coisa, a lista de contatos come metade do espaço e não existe botão nenhum pra escondê-la. A saída antiga era na mão: abrir o inspetor do navegador e mexer toda vez, ou conviver com a coluna ocupando espaço. A ideia foi virar esse "na mão" num botão que fica sempre lá.

O manifest, a identidade da extensão

Uma extensão não tem build nem instalação complicada. É uma pasta com um arquivo manifest.json e os arquivos de código. No navegador, você abre chrome://extensions, liga o modo do desenvolvedor e clica em "carregar sem compactação" apontando pra pasta. Está rodando.

O manifest.json é a identidade. O manifest_version é 3 (o formato atual); name e version aparecem na lista; e o action com default_popup é a janelinha que abre ao clicar no ícone. A primeira versão (0.1) não fazia nada útil: só abria um popup escrito "tá viva", de propósito, pra provar que ela carrega antes de botar lógica.

O content script

A parte que faz o trabalho é o content script: código seu que o navegador injeta dentro de uma página que não é sua. No manifest, o bloco content_scripts tem quatro campos. matches diz em quais páginas o script entra; js e css apontam os arquivos; run_at com document_idle roda depois que a página carregou.

"content_scripts": [
  {
    "matches": ["https://web.whatsapp.com/*"],
    "js": ["content.js"],
    "css": ["content.css"],
    "run_at": "document_idle"
  }
]

O content.js começa criando o botão. A função ensureButton monta um <button>, dá um id e o pendura no document.body, fora do pedaço que o WhatsApp controla. O CSS deixa ele position: fixed no canto inferior direito, com z-index altíssimo. No clique, alterna o estado e troca o próprio texto.

function ensureButton() {
    let btn = document.getElementById(BTN_ID);
    if (!btn) {
        btn = document.createElement("button");
        btn.id = BTN_ID;
        btn.addEventListener("click", function () {
            hidden = !hidden;
            apply();
        });
        document.body.appendChild(btn);
    }
    btn.textContent = hidden ? "Mostrar lista" : "Ocultar lista";
}

Esconder a coluna

A primeira tentativa furou. O WhatsApp Web tem um elemento com id side, que é a busca mais a lista. Esconder o #side sumiu com a lista, mas sobrou o cabeçalho "WhatsApp" em cima e o espaço vazio da coluna. O #side é só o miolo, não a coluna inteira.

A solução foi subir um nível. Em vez do #side, escondemos o pai dele (cabeçalho, largura e lista). Pra pegar esse pai sem saber o nome, usamos o seletor :has: "o elemento que tem um #side como filho direto". E ligamos isso por uma classe no <html>, não mexendo no elemento na mão. Fazer por seletor sobrevive quando o WhatsApp redesenha a tela.

html.wa-list-hidden :has(> #side) {
    display: none !important;
}

A linha estranha

Funcionou, mas sobrou uma linha vertical no meio da tela: a barra de redimensionar do WhatsApp, que mora fora da coluna. Pra matá-la, o content.js pega a faixa que segura a coluna e a conversa, olha os filhos diretos e procura um elemento fino (menos de 24px) e alto (mais de 40% da tela). Isso descreve uma divisória e nada mais. Não dependemos de um nome de classe que o WhatsApp pode trocar; dependemos do formato.

function hideDividers() {
    const side = document.querySelector("#side");
    const row = side && side.parentElement && side.parentElement.parentElement;
    if (!row) return;
    Array.from(row.children).forEach(function (child) {
        const r = child.getBoundingClientRect();
        if (r.width > 0 && r.width < 24 && r.height > window.innerHeight * 0.4) {
            child.dataset.waDivider = "1";
            child.style.setProperty("display", "none", "important");
        }
    });
}

Sobreviver ao React

O WhatsApp Web é feito em React, que redesenha pedaços da tela o tempo todo. Isso pode apagar o botão ou recriar a coluna do zero. O MutationObserver é uma ferramenta do navegador que avisa toda vez que o HTML muda. Observamos o document.body e, a cada mudança, garantimos que o botão existe e reaplicamos o estado. O requestAnimationFrame agrupa as mudanças num passo por quadro, pra não rodar mil vezes por segundo.

let raf = false;
new MutationObserver(function () {
    if (raf) return;
    raf = true;
    requestAnimationFrame(function () {
        raf = false;
        ensureButton();
        if (hidden) hideDividers();
    });
}).observe(document.body, { childList: true, subtree: true });

Resumo

Uma extensão é uma pasta com manifest.json carregada no modo desenvolvedor. O content script injeta o nosso código na página do WhatsApp. Escondemos a coluna pegando o pai do #side com :has e ligando por uma classe no <html>; matamos a divisória por formato (fina e alta); e usamos o MutationObserver pra sobreviver aos redesenhos do React. A coisa pra levar: extensão é só web (HTML, CSS, JS) rodando com um pouco mais de permissão sobre a página. De uma chateação de todo dia saiu um botão que fica sempre lá.
