O que é criar um novo módulo? +

7 arquivos — 3 novos + 4 edições — formam a cadeia para criar um módulo:

  • RootManager editar — Gerente Geral
    Uma linha é suficiente para o Jasap saber que o departamento existe.
  • DepartamentoManager novo — Gerente do Departamento
    Registra todas as actions e define os crachás de permissão. Prepara o andar antes de todo mundo chegar.
  • NovoModuloHome novo — Recepção
    Tela principal do módulo com sidebar e breadcrumb. É o primeiro lugar que o usuário vê ao entrar no departamento.
  • NovoModuloMdFactory novo — Ferramentaria
    Fabrica os models sob encomenda e guarda pra reusar. As ferramentas que as actions usam para acessar o banco de dados.
  • AppsRootModelFactory editar — Ferramentaria Central
    Conecta a ferramentaria do módulo ao sistema. Sem isso, nenhum outro departamento acessa suas ferramentas.
  • PnlCfg editar — Cabine do Avião
    Painel com 500 botões. Plugamos um cabo e saímos de fininho — adiciona o checkbox para ativar/desativar o módulo.
  • HomeDesk editar — Saguão de Entrada
    Adiciona o card do módulo no menu principal. Só aparece se o interruptor do PnlCfg está ligado.
Recepcao do departamento.

Monta o menu lateral (sidebar) com os guiches (MID) e o breadcrumb de navegacao. E o primeiro lugar que o usuario ve ao entrar no modulo.

Cada guiche (MID) aponta para uma entidade — Pessoa, Produto, Perfil. O MenuInicial monta o painel com os guiches disponiveis e o MenuItem redireciona o usuario ao clicar.

Gerente do departamento.

Registra todas as actions (funcionarios) no framework via regAction e define os crachas de permissao via regFun. Prepara o andar antes de todo mundo chegar.

Estende PnlManager — herda o registro de permissoes e a integracao com o painel de controle. Sem ele, nenhuma action do modulo e encontrada pelo Jasap.

Ferramentaria do departamento.

Fabrica os models sob encomenda e guarda pra reusar (lazy initialization). As ferramentas que as actions usam para acessar o banco.

Cada metodo (pesModel, proModel, prfModel) cria o model na primeira chamada e retorna o mesmo nas proximas. Padrao protected — so o proprio modulo acessa.

Ferramentaria central.Conecta a ferramentaria do modulo ao sistema. Sem essa linha, nenhum outro departamento consegue acessar suas ferramentas via getFactory().

Uma unica linha: criar o campo protected e o metodo publico. Mesmo padrao para todos os modulos — Laboratorio, Contatos, Painel, Ged, Aula.
